縦書き html/PDF/EPUB 置き場

縦書き html、PDF、EPUB の置き場です。主に自分用です（組版好きなので）。

名義について

  • 内容的に地雷系が多く不快感を与えかねないのと、そもそもグレーな活動なので、一応別名義でやっています。
  • ただし、自分から宣伝しないだけで別に隠しているわけではありませんので、普通に言及いただいてかまいませんし、呼ばれれば答えます。何かあれば pixiv DM か Twitter へご連絡ください。

フォーマットについて

組版

  • 以下の環境で typeset しています。2022 年に TATEditor から試験的に乗り換えました。
    • 執筆 + html 化: VSCode + novel-writer
    • バージョン管理: GitHub (private repositories)
    • CSS 組版: vivliostyle CLI + Chromium
    • EPUB 化: Sigil
    • 表紙作成: かんたん表紙メーカー (サイズが合っていません。すいません)

html

  • html は Firefox では縦中横が表示されません。

EPUB

  • EPUB は以下の環境で動作確認しています。Google Books、読書尚友および Kindle for iOS では正しく表示できません。
    • ○ Apple Books (iOS, MacOS)
    • ○ honto (iOS)
    • ○ 楽天 Kobo (iOS, Android, MacOS)
    • ○ BOOK☆WALKER (iOS、MacOS)
    • ○ Kinoppy (iOS, Android, MacOS)
    • ✕ Kindle (iOS): 横書きになる、ルビが正しく表示されない
    • ○ Kindle (Android)
    • ○ Bibi (WebUI)
    • ○ Sigli (MacOS)
    • ○ Calibre (MacOS)
    • △ Google Books (Android): 縦中横が正しく表示されない、ダーシがセンタリングされていない
    • ✕ Google Books (WebUI): 2 ページ目以降が表示されない
    • △ Perfect Viewer (Android): フォントによっては一部の約物（全角「：」等）が横書きになる（Noto Serif 等）
    • △ エア草紙 (WebUI, MacOS): ブラウザによっては長音記号の表示がおかしい。二重山括弧が強制的にルビになってしまう
    • ✕ 読書尚友 (iOS): 空行、全角空白が無視される
    • ✕ 読書尚友 Lite (Android): 空行が無視される。ダーシ、三点リーダ等の表示がおかしい
  • 個人的には honto と楽天 Kobo での見た目が好きです。

PDF

  • PDF のサイズは四六判から文庫サイズ (A6) に変更しました。印刷すると少し文字が大きすぎるかもしれません。電子的閲覧向けです。
